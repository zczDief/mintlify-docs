# Changelog

Acompanhe todas as atualizações, melhorias e correções da Judit API.

## [v2.1.0] - 2024-01-15

### ✨ Novidades
- **Tracking Avançado**: Configurações personalizadas de frequência de monitoramento
- **Filtros Inteligentes**: Novos filtros por tipo de movimento processual
- **SDK Python v2.1**: Suporte completo a async/await
- **Webhook Retry**: Retry automático para webhooks que falharam

### 🚀 Melhorias
- **Performance**: Consultas 40% mais rápidas em média
- **Cache Inteligente**: Cache automático para consultas repetidas
- **Rate Limiting**: Limites mais flexíveis para planos Enterprise
- **Documentação**: Nova seção de exemplos práticos

### 🐛 Correções
- Corrigido timeout em consultas do TJSP durante picos de tráfego
- Resolvido problema de encoding em anexos com caracteres especiais
- Corrigida paginação inconsistente em listas grandes de processos

### 🔧 Alterações Técnicas
- Migração para nova infraestrutura de alta disponibilidade
- Implementação de circuit breakers para tribunais instáveis
- Otimização de queries no banco de dados

### 🎉 Release Principal
- **Nova Arquitetura**: Microserviços independentes por funcionalidade
- **Vault Seguro**: Sistema criptografado para credenciais de tribunais
- **Tracking em Tempo Real**: Monitoramento contínuo de processos
- **API Unificada**: Endpoints consistentes para todos os recursos

### ✨ Novidades
- **Consultas por OAB**: Busque processos por número de advogado
- **Anexos Automáticos**: Download automático de documentos
- **Webhooks Avançados**: Payload estruturado com metadados
- **Dashboard Analytics**: Métricas de uso em tempo real

### 🚀 Melhorias
- **Uptime 99.9%**: Nova infraestrutura de alta disponibilidade
- **Latência Reduzida**: Tempo de resposta 60% menor
- **Escalabilidade**: Suporte a 10x mais requisições simultâneas
- **Segurança**: Criptografia end-to-end para dados sensíveis

### ⚠️ Breaking Changes
- **Endpoints Antigos**: URLs v1.x descontinuadas (migração obrigatória)
- **Formato de Resposta**: Nova estrutura JSON padronizada
- **Autenticação**: API Keys antigas precisam ser renovadas
- **Rate Limits**: Novos limites por plano de assinatura

### 📚 Documentação
- **Guia de Migração**: Passo a passo para atualizar integração

## Política de Versionamento

### 📋 Semantic Versioning
Seguimos o padrão [SemVer](https://semver.org/):
- **MAJOR**: Breaking changes que requerem alterações no código
- **MINOR**: Novas funcionalidades compatíveis com versão anterior
- **PATCH**: Correções de bugs e melhorias menores

### 🔄 Ciclo de Release
- **Major Releases**: 2x por ano (Junho e Dezembro)
- **Minor Releases**: Mensalmente
- **Patch Releases**: Conforme necessário (bugs críticos)

### ⚠️ Deprecation Policy
- **Aviso Prévio**: 90 dias para funcionalidades descontinuadas
- **Suporte Estendido**: 6 meses após deprecation
- **Migração Assistida**: Guias e ferramentas de migração
